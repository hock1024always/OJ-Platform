Step 1 - 填写题目基本信息

填写题目的标题、描述、难度等基本信息。描述需要包含题意说明和输入输出示例，让做题者能理解题目要求。

标题：简洁明了，通常为算法名称或问题概括

两数之和

描述：包含题意、约束条件、示例输入输出。建议格式如下：

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那两个整数，并返回它们的数组下标。 你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。 你可以按任意顺序返回答案。 示例 1： 输入：nums = [2,7,11,15], target = 9 输出：[0,1] 示例 2： 输入：nums = [3,2,4], target = 6 输出：[1,2] 提示： - 2 <= nums.length <= 10^4 - -10^9 <= nums[i] <= 10^9

难度：Easy / Medium / Hard

标签：多个标签用英文逗号分隔，如 数组,哈希表、链表,双指针、动态规划

时间/内存限制：默认值（5000ms / 256MB）适用于大多数题目。如果题目需要高效算法，可以适当降低。

Step 2 - 定义函数签名

函数签名决定了做题者看到的代码模板，以及系统如何读取测试数据、调用用户函数。 你只需要定义函数名、参数列表和返回类型，系统会自动生成 Go / C++ / C / Java 四种语言的模板和驱动代码。

函数名：驼峰命名，与 LeetCode 风格一致。例如：twoSum、maxProfit、reverseList

参数：每个参数需要填写「参数名」和「类型」。系统支持以下类型：

类型说明测试数据格式（输入/输出各占一行）适用场景举例
int整数42爬楼梯的 n
float浮点数3.14数学计算
string字符串hello world（整行）无重复字符的最长子串
bool布尔值true 或 false-
[]int整数数组2 7 11 15（空格分隔）两数之和的 nums
[]string字符串数组eat tea tan ate（空格分隔）字母异位词分组
[][]int二维整数数组第一行为行数，之后每行一行数据岛屿数量的 grid
[]byte字符数组abcabc（整行字符串）字符频率统计
ListNode链表1 2 3 4 5（空格分隔节点值）反转链表
TreeNode二叉树1 2 3 null 5（层序遍历，null 表示空）二叉树的中序遍历

返回类型：与参数类型相同的选项列表，另外可选 (void) 表示无返回值。

常见题目签名示例：

Step 4 - 添加测试用例

测试用例用于判定做题者的代码是否正确。有两种方式添加：
  • 手动添加：在下方直接填写输入和期望输出。发布前至少添加 2~3 个公开用例（做题者可见）。
  • 自动生成：粘贴标程代码（标准答案），系统自动随机生成输入、运行标程得到输出，批量创建测试用例。如果题目尚未发布，系统会自动先发布题目。

测试数据的输入和输出都是纯文本，每个参数占一行，格式与第2步中选择的类型对应：

示例 - 两数之和（参数：nums []int, target int；返回：[]int）

输入（两行，第一行是 nums，第二行是 target）： 2 7 11 15 9 期望输出（一行，返回的两个下标）： 0 1

示例 - 反转链表（参数：head ListNode；返回：ListNode）

输入（链表节点值，空格分隔）： 1 2 3 4 5 期望输出（反转后的链表）： 5 4 3 2 1

示例 - 二叉树最大深度（参数：root TreeNode；返回：int）

输入（层序遍历，null 表示空节点）： 3 9 20 null null 15 7 期望输出： 3

示例 - 岛屿数量（参数：grid [][]byte；返回：int）

输入（第一行是行数，之后每行是 grid 的一行）： 4 11110 11010 11000 00000 期望输出： 1

自动生成说明：

  • 粘贴你的 Go 标准答案（只写函数体，不需要 main 函数和 import），点击「自动生成」即可。
  • 如果题目尚未发布，系统会自动先发布题目再生成测试数据，无需手动操作。
  • 生成数量：建议 50 个。系统会随机生成输入数据，运行标程获得正确输出。
  • 公开数量：前 N 个用例会标记为「公开」，做题者可以在题目页面看到。建议 2~3 个。
  • 生成后可点击「验证测试用例」确认标程在所有用例上全部通过。

标程代码示例（两数之和）：

func twoSum(nums []int, target int) []int { hashMap := make(map[int]int) for i, v := range nums { if j, ok := hashMap[v]; ok { return []int{j, i} } hashMap[target-v] = i } return nil }
