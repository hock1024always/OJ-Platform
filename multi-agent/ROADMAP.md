## 第一阶段：Agent沙箱增强（利用现有能力）

### 1.1 安全增强
- [ ] 在go-judge基础上增加PII脱敏能力（参考AKIOS）
- [ ] 敏感数据检测与过滤
- [ ] API Key 安全存储与轮换

### 1.2 资源控制
- [ ] 增加预算/配额控制（限制单个Agent可消耗的资源/API费用）
- [ ] Token 使用量统计与限制
- [ ] 并发任务数限制

### 1.3 审计日志
- [ ] 增加操作审计日志（所有Agent执行记录上链/防篡改）
- [ ] 执行历史回放
- [ ] 异常行为检测

## 第二阶段：多Agent编排能力

### 2.1 DAG工作流引擎
- [ ] 设计DAG工作流数据结构（节点=Agent，边=数据流）
- [ ] 工作流解析与执行
- [ ] 条件分支与循环支持
- [ ] 错误处理与重试机制

### 2.2 Agent间通信
- [ ] 实现Agent间通信机制（参考A2A协议或使用NATS）
- [ ] 消息路由与广播
- [ ] 请求-响应模式
- [ ] 发布-订阅模式

### 2.3 可视化编排
- [ ] 开发可视化工具体（可用React Flow快速搭建）
- [ ] 拖拽式Agent编排
- [ ] 工作流模板库

## 第三阶段：集成与可视化

### 3.1 实时通信
- [ ] 集成WebSocket实时推送Agent执行状态（已有Centrifuge，正好复用）
- [ ] 实时进度更新
- [ ] 多客户端同步

### 3.2 会话管理
- [ ] 实现对话/会话回放功能（参考Agensight）
- [ ] 会话持久化
- [ ] 会话导出与分享

### 3.3 LLM集成
- [ ] 对接主流LLM（OpenAI/Anthropic/本地Ollama）
- [ ] 多模型切换
- [ ] 模型能力评估
